# Why This Is A Non-Wager Civic Market

The forecast side is an information signal. It helps people see expectations, uncertainty, and resolution assumptions around a civic issue.

The reward side is separate. A sponsor funds useful civic participation, and the receipt path releases reward units only after the action receipt is verified.

This separation matters:

- Forecast participation does not create an outcome payout claim.
- Sponsor rewards are tied to verified civic action, not correct prediction.
- The public ledger and `npm run civic:verify-receipt` show what the Solana receipt proves and what still needs an external civic data source.

Phase 3 keeps the boundary visible in the UI, API responses, proof packets, SDK wrappers, and independent verifier output.
